Backup plan: SQLite threading / connection lifecycle

If the per-request async dependency approach ever fails or is not possible:

Option A: Single-worker mode
- Run uvicorn with a single worker to avoid cross-thread usage.
  Example: uvicorn web.app:app --host 127.0.0.1 --port 8000 --workers 1

Option B: Keep per-request connections but allow cross-thread use
- Set sqlite3.connect(..., check_same_thread=False) and ensure each request still
  creates and closes its own connection (never share connections globally).

Option C: Move to aiosqlite
- Replace sqlite3 with aiosqlite and make all DB access async.
- This removes threadpool handoff issues and keeps DB work on the event loop.

Option D: External DB
- If concurrency grows, move to Postgres (or another server DB) and update the
  repository layer accordingly.
